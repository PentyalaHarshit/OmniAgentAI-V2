# Advanced Algorithm RAG Knowledge Base

Algorithm: dijkstra
Keywords: dijkstra, shortest path, weighted graph, priority queue, min heap
Category: graph
Use When: Need shortest paths from one source in a graph with non-negative edge weights.
Approach:
Use an adjacency list and a min-priority queue. Relax edges and skip outdated queue entries.
Time Complexity: O((V + E) log V)
Memory Complexity: O(V + E)

Algorithm: zero_one_bfs
Keywords: 0-1 bfs, zero one bfs, weight 0 1, deque shortest path
Category: graph
Use When: Need shortest paths where every edge weight is either 0 or 1.
Approach:
Use a deque. Push front after a 0-weight relaxation and push back after a 1-weight relaxation.
Time Complexity: O(V + E)
Memory Complexity: O(V + E)

Algorithm: segment_tree
Keywords: segment tree, range query, range sum, range max, point update, range update
Category: data structure
Use When: Need range queries with updates.
Approach:
Build a binary tree over array intervals. Recompute parent values after updates and combine covered intervals during queries.
Time Complexity: O(log N) per query/update
Memory Complexity: O(N)

Algorithm: lazy_segment_tree
Keywords: lazy segment tree, range update, lazy propagation, range add, range assign
Category: data structure
Use When: Need many range updates and range queries.
Approach:
Store pending updates in lazy nodes and push them only when descending into children.
Time Complexity: O(log N) per query/update
Memory Complexity: O(N)

Algorithm: fenwick_tree
Keywords: fenwick tree, binary indexed tree, prefix sum, point update
Category: data structure
Use When: Need prefix sums or invertible range sums with point updates.
Approach:
Use index lowbit jumps for update and prefix query.
Time Complexity: O(log N) per query/update
Memory Complexity: O(N)

Algorithm: lowest_common_ancestor_binary_lifting
Keywords: lca, lowest common ancestor, binary lifting, tree ancestor, kth parent
Category: tree
Use When: Need fast ancestor or LCA queries in a static tree.
Approach:
Run DFS from the root. Store depth and up[node][j] as the 2^j-th ancestor. Lift the deeper node, then lift both nodes together.
Time Complexity: O(N log N) preprocessing, O(log N) per query
Memory Complexity: O(N log N)

Algorithm: heavy_light_decomposition
Keywords: hld, heavy light decomposition, tree path query, path maximum, path sum
Category: tree
Use When: Need path queries on a tree with updates.
Approach:
Compute subtree sizes, choose heavy children, decompose into chains, and use a segment tree over node positions.
Time Complexity: O(log^2 N) per path query/update
Memory Complexity: O(N)

Algorithm: centroid_decomposition
Keywords: centroid decomposition, tree distance queries, nearest marked node, dynamic tree queries
Category: tree
Use When: Need dynamic distance queries on a tree.
Approach:
Recursively split the tree at centroids and store/update answers along centroid ancestors.
Time Complexity: O(N log N) preprocessing, O(log N) or O(log^2 N) per operation
Memory Complexity: O(N log N)

Algorithm: dsu_on_tree
Keywords: dsu on tree, small to large, subtree queries, sack technique
Category: tree
Use When: Need aggregate statistics for every subtree.
Approach:
Process light children temporarily, keep the heavy child's data, then merge light data into the kept structure.
Time Complexity: O(N log N) or O(N) depending on counter operations
Memory Complexity: O(N)

Algorithm: mo_algorithm
Keywords: mo algorithm, sqrt decomposition, offline queries, range queries
Category: offline queries
Use When: Need answer many static range queries with add/remove transitions.
Approach:
Sort queries by block and right endpoint, then move a sliding window while maintaining the answer.
Time Complexity: O((N + Q) sqrt N)
Memory Complexity: O(N)

Algorithm: convex_hull_trick
Keywords: convex hull trick, cht, li chao tree, dp optimization
Category: dynamic programming
Use When: DP transitions can be represented as minimum or maximum queries over lines.
Approach:
Insert lines and query by x-coordinate using a Li Chao Tree or monotonic hull.
Time Complexity: O(N log C)
Memory Complexity: O(N)

Algorithm: divide_conquer_dp
Keywords: divide and conquer dp, dp optimization, monotonic opt
Category: dynamic programming
Use When: DP optimal split points are monotonic.
Approach:
Compute each layer recursively and restrict the search interval for the optimal split.
Time Complexity: O(KN log N) or O(KN)
Memory Complexity: O(NK) or O(N)
